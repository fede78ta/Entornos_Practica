1. Qué es HTML, cuando fue creado, cuáles fueron las distintas versiones y cuál es la última?
    HTML es un lenguaje de marcado el cual se utiliza en la creacion de las paginas web y fue creado en el año 1991.
Las distintas versiones de HTML fueron HTML 2.0, el HTML 3.2, el HTML 4.0, el HTML 4.01  y su ultima version es el HTML 5.

2.b)
<div id="bloque1">Contenido del bloque1</div>

    La etiqueta div sirve para definir las divisiones lógicas existentes en el contenido de una página web.
    La etiqueta div va en la seccion body de un archivo HTML.
    El atributo id no es obligatorio y sirve para identificar el div en CSS y poder modificarlo.

2.c)
<img src="" alt="lugar imagen" id="im1" name="im1" width="32" height="32" longdesc="detalles.htm" />

    La etiqueta img sirve para poner una imagen en la pagina web.

    Atributos de img
    height sirve para definir la altura de la imagen.
    width sirve para definir la anchura de la imagen.
    longdesc complementa al texto de alt y se emplea para ofrecer una descripción más larga del elemento, no es obligatorio.
    name sirve para ponerle un nombre a la imagen
    id sirve para identificar el elemento en CSS y poder modificarlo.
    alt este atributo define el texto alternativo que describe la imagen, texto que los usuarios verán en el caso que la imagen no aparesca.
    src es la url para la imagen a utilizar y es un atributo obligatorio.

2.e)
<a href="http://www.e-style.com.ar/resumen.html" type="text/html" hreflang="es" charset="utf-8" rel="help">Resumen HTML </a>

    La etiqueta sirve para crear un enlace a otras páginas de internet, archivos o ubicaciones dentro de la misma página, direcciones de correo, o cualquier otra URL.

Atributos
href Contiene una URL o un fragmento de URL que determina el destino del enlace.
type especifica el tipo de medio (media type) en la forma de MIME type (en-US) para la URL enlazada.
hreflang este atributo indica el lenguaje humano del recurso al que se enlaza.
charset este atributo define la codificación de caracteres (character encoding) de la URL enlazada y no es obligatorio.
rel especifica la relación del objeto de destino con el objeto de enlace.
